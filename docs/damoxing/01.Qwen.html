Qwen（通义千问）

1. 概述与发展历史

1.1 简介

Qwen（通义千问）是阿里巴巴集团自主研发的大语言模型系列，由阿里云DAMO Academy（达摩院）开发。Qwen系列涵盖了从轻量级到超大规模的语言模型，以及多模态理解模型，是当前最具影响力的中文大模型之一。

Qwen的核心优势在于：

  • 强大的中文理解与生成能力：基于海量中文数据预训练，在中文任务上表现优异
  • 开源开放：Qwen2.5系列全面开源，支持本地部署
  • 多模态支持：提供视觉(Qwen-VL)和语音(Qwen-Audio)模型
  • 生态完善：配套API、SDK、本地部署方案一应俱全

1.2 发展历程

时间版本主要特性
2023年4月Qwen-1首次发布，7B/14B参数版本
2023年9月Qwen-1.5训练稳定性提升，支持更长的上下文
2024年6月Qwen-2全面升级，GQA注意力机制，代码能力大幅提升
2024年9月Qwen-2.5发布32B/72B超大杯，MoE架构探索
2024年11月Qwen-2.5-VL视觉理解能力显著增强
2025年1月Qwen-2.5-Math数学推理专项优化
2025年3月Qwen-3（预览）新一代架构，思考链能力增强

1.3 母公司背景

阿里巴巴集团是中国最大的电子商务公司之一，阿里云是其云计算部门。Qwen模型通过阿里云平台提供服务，与阿里巴巴的电商、云计算生态系统深度整合。

3. 使用方式详解

3.1 API调用

Qwen API基于HTTP RESTful协议，支持同步和流式两种调用方式。

3.1.1 同步调用

请求端点：

POST https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation

Python示例：

import requests

url = "https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation"

headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

payload = {
    "model": "qwen-turbo",  # 可选: qwen-plus, qwen-max, qwen-max-longcontext
    "input": {
        "messages": [
            {"role": "system", "content": "你是一个专业的AI助手。"},
            {"role": "user", "content": "请用Python写一个快速排序算法。"}
        ]
    },
    "parameters": {
        "temperature": 0.7,
        "top_p": 0.8,
        "max_tokens": 2048,
        "result_format": "message"
    }
}

response = requests.post(url, headers=headers, json=payload)
print(response.json())

from dashscope import Generation

response = Generation.call(

if response.status_code == 200:
    print(response.output.choices[0].message.content)
else:
    print(f"Error: {response.code} - {response.message}")

# 流式调用
from dashscope import Generation
from dashscope.api_entities.dashscope_response import GenerationResponse

def print_stream_response(res):
    if res.status_code == 200:
        for chunk in res.output.choices:
            print(chunk.message.content, end='', flush=True)
        print()
    else:
        print(f"Error: {res.code} - {res.message}")

response = Generation.call(
    model=Generation.Models.qwen_plus,
    messages=[
        {"role": "user", "content": "写一首关于春天的诗。"}
    ],
    stream=True,
    stream_callback=print_stream_response
)

3.2.2 Java SDK

import com.alibaba.dashscope.DashScope;
import com.alibaba.dashscope.base.FunctionType;
import com.alibaba.dashscope.common.Message;
import com.alibaba.dashscope.common.Role;
import com.alibaba.dashscope.generation.Generation;
import com.alibaba.dashscope.generation.GenerationParam;
import com.alibaba.dashscope.generation.GenerationResult;

import java.util.Arrays;

public class QwenExample {
    public static void main(String[] args) {
        Generation gen = new Generation();

        Message systemMsg = Message.builder()
                .role(Role.SYSTEM.getValue())
                .content("你是一个乐于助人的AI助手。")
                .build();

        Message userMsg = Message.builder()
                .role(Role.USER.getValue())
                .content("什么是机器学习？")
                .build();

        GenerationParam param = GenerationParam.builder()
                .model(Generation.Models.QWEN_TURBO.getValue())
                .messages(Arrays.asList(systemMsg, userMsg))
                .temperature(0.7)
                .maxTokens(2048)
                .build();

        GenerationResult result = gen.call(param);
        System.out.println(result.getOutput().getChoices().get(0).getMessage().getContent());
    }
}

5. 优缺点分析

5.1 优势

5.1.1 中文能力突出

  • 基于海量中文语料预训练
  • 中文理解和生成质量业内领先
  • 深入理解中国文化和语境

5.1.2 开源生态完善

  • Qwen2.5系列全面开源（Apache 2.0许可证）
  • 提供从0.5B到72B全系列模型
  • 社区活跃，生态繁荣

5.1.3 多模态能力强

  • Qwen2.5-VL：强大的视觉理解能力
  • Qwen-Audio：音频分析处理
  • 端到端多模态任务支持

5.1.4 性价比高

  • Turbo版本价格极具竞争力
  • 开源版本可免费本地部署
  • 推理效率优化（vLLM支持）

5.1.5 阿里云生态集成

  • 与阿里云其他服务无缝集成
  • 企业用户友好（SLA保障、票据支持）
  • 安全合规（中国境内数据合规）

5.2 劣势

5.2.1 英文能力相对弱

  • 英文任务表现略逊于GPT-4
  • 部分英文专业术语处理不够准确
  • 跨语言任务需要指定语言

5.2.2 超长上下文支持有限

  • 标准版本支持32K上下文
  • 超长上下文版本价格较高
  • 长文本任务偶有信息丢失

5.2.3 创意写作多样性

  • 部分用户反映输出较为保守
  • 文学创作想象力有待提升
  • 需要调整temperature获得更创意输出

5.2.4 数学推理仍有差距

  • 复杂数学问题不如Claude和GPT-4
  • Qwen2.5-Math专项优化但仍有提升空间
  • 证明类任务表现一般

5.2.5 国际市场覆盖

  • 主要面向中文用户优化
  • 海外数据中心覆盖有限
  • 国际化功能更新滞后

modelstring是-模型名称
inputobject是-输入内容
parametersobject否见下方生成参数

9.3.2 input字段结构

{
  "messages": [
    {
      "role": "system",

role可选值：

  • system：系统角色设定
  • user：用户输入
  • assistant：助手回复

9.3.3 parameters参数

参数名类型默认值取值范围说明
temperaturefloat0.70-2采样温度，越高越随机
top_pfloat0.80-1Nucleus采样
top_kint50正整数Top-K采样
max_tokensint20481-6000最大生成token数
repetition_penaltyfloat1.00-2重复惩罚
result_formatstring"message"message/text返回格式
stopstring/arraynull-停止词
streambooleanfalse-是否流式输出
enable_searchbooleanfalse-启用联网搜索

场景推荐模型原因

10.2.1 系统提示词优化

# 不推荐
"你是一个AI"

# 推荐
"""你是一位拥有10年经验的资深Python后端工程师，擅长Flask和Django框架。
回答时要：
1. 代码遵循PEP8规范
2. 附上必要的注释说明
3. 提供使用示例
4. 指出可能的坑和最佳实践"""

10.2.2Few-shot示例

messages = [
    {"role": "system", "content": "你是一个翻译专家。"},
    {"role": "user", "content": "翻译以下句子为英文：我爱你"},
    {"role": "assistant", "content": "I love you."},
    {"role": "user", "content": "翻译以下句子为英文：今天天气真好"}
]

10.3 性能优化

  1. 使用流式输出：减少用户等待感知
  2. 设置合理的max_tokens：避免不必要的计算
  3. 缓存常用提示词：减少token消耗
  4. 批量处理：多个请求合并处理
