Gemini AI

1. 概述与发展历程

公司背景

Google DeepMind 是 Google 旗下专注于人工智能研究的部门，由 Google Brain 和 DeepMind 合并而成。Gemini 是 Google DeepMind 打造的多模态 AI 模型，是 Google 官方推荐的 GPT 替代方案，与 Google 生态（搜索、Workspace、云服务）深度集成。

发展历程

时间里程碑
2023 年 12 月Gemini 1.0 发布（Ultra/Pro/Nano）
2024 年 2 月Gemini 1.5 发布，支持 1M token 上下文
2024 年 5 月Gemini 1.5 Pro 更新，上线 Google AI Studio
2024 年 8 月Gemini 2.0 发布，性能大幅提升
2025 年Gemini 2.5 发布，超长上下文优化
2026 年Gemini 3 发布，多模态能力进一步增强

核心产品线

  • Gemini Ultra：旗舰版，最强能力
  • Gemini Pro：平衡版，商业可用
  • Gemini Flash：轻量版，快速响应
  • Gemini Nano：设备端模型

2. 开放平台与访问入口

官方网站

  • Google AI Studio：https://aistudio.google.com/ （开发平台）
  • Gemini API：https://ai.google.dev/ （API 服务）
  • Google Cloud：https://cloud.google.com/vertex-ai/ （企业级）

第三方接入

  • SiliconFlow：国内 API 中转
  • One API：开源 API 管理
  • FastGPT：知识库平台

应用入口

  • Gemini 网页版：https://gemini.google.com/
  • Google App：Gemini 助手集成在 Google App
  • Bard：已整合为 Gemini

3. 使用方式详解

API 调用

Python SDK

import google.genai as genai

# 初始化客户端
client = genai.Client(api_key="YOUR_API_KEY")

# 同步调用
response = client.models.generate_content(
    model="gemini-2.0-flash",
    contents="请解释什么是 RAG 技术"
)
print(response.text)

Google Gen AI SDK (新版本)

from google import genai

client = genai.Client(api_key="YOUR_API_KEY")

# 简单调用
response = client.models.generate_content(
    model="gemini-2.0-flash",
    contents="用 Python 写一个快速排序"
)
print(response.text)

# 流式输出
for chunk in client.models.generate_content_stream(
    model="gemini-2.0-flash",
    contents="写一首关于海的诗"
):
    print(chunk.text, end="", flush=True)

cURL 调用

curl -X POST "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [{
      "parts": [{"text": "解释什么是向量数据库"}]
    }]
  }'

本地部署

Gemini 部分模型支持本地部署：

Google AI Edge

// 在 Web 中使用 Gemini Nano
import { GeminiNano } from "@ai/gemini-nano";

const model = await GeminiNano.load();
const result = await model.generate("你好");

5. 优缺点分析

优势

  1. 免费模型强：Gemini 2.0 Flash 完全免费且能力很强
  2. 超长上下文：1.5/2.0 Pro 支持 1M token 上下文
  3. 多模态领先：原生多模态设计，图像/视频/音频统一处理
  4. Google 生态：与 Google 搜索、云服务、Workspace 深度集成
  5. 价格实惠：相比 OpenAI 有明显价格优势
  6. 开发工具：Google AI Studio 提供完善的开发工具

劣势

  1. 国内访问：API 在中国访问不稳定
  2. 中文能力：中文能力不如英文
  3. 长文本生成：复杂长文本生成有时不够稳定
  4. Function Calling：工具调用能力比 Claude 稍弱
  5. 文档质量：部分功能的文档不如 OpenAI 完善

6. 擅长领域

  1. 多模态理解

    • 图像/视频/音频统一处理
    • 文档理解与分析
    • 图表解读

  2. 长上下文任务

    • 整本书籍总结
    • 长视频分析
    • 代码库理解

  3. Google 生态集成

    • 搜索增强
    • Workspace 文档处理
    • BigQuery 数据分析

  4. 免费应用

    • 日常对话
    • 简单问答
    • 原型开发

7. 多模态能力

Gemini 是原生多模态模型，设计时就将图像、视频、音频统一处理：

支持能力

  • 图像理解：图表、文档、照片、截图
  • 视频理解：视频内容分析、帧提取
  • 音频处理：语音识别、音频分析
  • 文档处理：PDF、Word、Excel

多模态示例

# 图像理解
response = client.models.generate_content(
    model="gemini-2.0-flash",
    contents=[
        "请描述这张图片的内容",
        {"inline_data": {"mime_type": "image/png", "data": base64_image}}
    ]
)

# 视频理解（需要上传到 Google Cloud Storage）
response = client.models.generate_content(
    model="gemini-1.5-pro",
    contents=[{
        "file_data": {
            "mime_type": "video/mp4",
            "file_uri": "gs://bucket/video.mp4"
        }
    }, {"text": "这个视频的主要内容是什么？"}]
)

9. API 端点与参数

基础端点

POST https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent

请求格式

response = client.models.generate_content(
    model="gemini-2.0-flash",
    contents="你的问题",
    config={
        "temperature": 0.9,
        "max_output_tokens": 2048,
        "top_p": 0.95,
    }
)

请求参数

参数类型说明
contentsarray输入内容
generation_configobject生成配置
safety_settingsarray安全设置
system_instructionstring系统指令

generation_config

参数类型默认值说明
temperaturefloat0.9采样温度
top_pfloat1核采样
top_kint-top-k 采样
max_output_tokensint8192最大输出
stop_sequencesarray-停止序列

10. 最佳实践

模型选型

# 日常对话 / 原型 -> Gemini 2.0 Flash (免费)
# 复杂任务 / 长文档 -> Gemini 2.5 Pro
# 简单任务 / 客服 -> Gemini 2.0 Flash-Lite
# 成本优先 -> Gemini 1.5 Flash

Prompt 技巧

  1. 结构化输入：使用清晰的格式
  2. 指定输出格式：明确要求 JSON 或其他格式
  3. 分步骤：复杂任务分多个请求
  4. 安全设置：敏感场景配置安全过滤

成本优化

# 1. 优先使用免费模型
# 2. Gemini 2.0 Flash 满足大多数场景
# 3. 控制 max_output_tokens
# 4. 批量处理请求
