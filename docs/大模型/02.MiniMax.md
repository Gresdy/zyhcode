# MiniMax 大模型

## 1. 概述与发展历程

### 1.1 公司简介

MiniMax（稀宇科技）是一家成立于 2021 年的人工智能科技公司，由前商汤科技副总裁闫俊杰创立，总部位于中国。MiniMax 是国内最早专注于通用人工智能技术研发的初创公司之一，核心团队来自微软、谷歌、百度等顶级科技公司。

MiniMax 的战略定位是"**端到端的通用人工智能平台**"，致力于研发基于混合专家（Mixture of Experts，MoE）架构的大语言模型，并在文本、语音、视频等多模态领域进行全面布局。公司于 2024 年发布了多款重磅产品，包括对话应用 MiniMax 和 AI 视频生成应用海螺AI。

### 1.2 技术路线与特色

MiniMax 的核心技术特色是 **MoE（混合专家）架构**。与传统的 Dense 模型不同，MoE 架构将模型能力分散到多个"专家"网络中，每次推理时只激活与当前任务相关的专家，从而在保持模型总参数量的同时大幅降低实际计算量。

这种架构的优势包括：

- **高效推理**：实际激活参数远小于模型总参数，以更低的计算成本实现同等能力
- **可扩展性强**：可以通过增加专家数量来扩展模型能力，无需显著增加推理成本
- **多任务适应**：不同专家擅长不同类型的任务，模型整体具备更强的多任务泛化能力

### 1.3 发展里程碑

| 时间 | 事件 |
|------|------|
| 2021 年 | MiniMax 成立，开始研发通用 AI 技术 |
| 2023 年 | 发布 abab 系列早期版本 |
| 2024 年初 | 发布 abab6 及 abab6.5 系列，MoE 架构成熟 |
| 2024 年中 | 推出 MiniMax 对话应用和海螺AI 视频生成应用 |
| 2025 年 | 持续迭代模型能力，推出更大规模的 MoE 模型 |
| 2026 年 | 发布 abab6.5s 等优化版本，支持更长的上下文 |

## 2. 模型系列详解

### 2.1 主要模型型号

MiniMax 目前提供以下主要模型系列：

#### abab6.5s

abab6.5s 是 MiniMax 在 2024 年底至 2025 年初推出的主力模型，定位为**高性价比的旗舰级文本模型**。

**核心参数：**

- **总参数量**：约 2000 亿（200B）参数
- **激活参数量**：约 200 亿（20B）参数（MoE 架构）
- **上下文长度**：支持最长 256K tokens（约 20 万汉字）
- **上下文窗口**：原生支持 128K，可扩展至 256K

**能力特点：**

- 中英文双语能力出色
- 代码生成与推理能力强
- 支持函数调用（Function Calling）
- 支持结构化输出（JSON Mode）
- 长上下文理解与检索能力强

#### abab6.5

abab6.5 是 abab6.5s 的增强版本，拥有更高的模型总参数量和更强的综合能力。

**核心参数：**

- **总参数量**：约 3000 亿（300B）参数
- **激活参数量**：约 300 亿（30B）参数
- **上下文长度**：支持最长 1M tokens（约 80 万汉字）

#### 其他专用模型

MiniMax 还提供针对特定场景优化的专用模型：

- **embedding 模型**：用于文本向量化，支持 512 维和 1536 维输出
- **语音合成模型**：支持多语言、多音色的语音生成
- **视频生成模型**：通过海螺AI 产品对外提供服务

### 2.2 多模态能力

**MiniMax 是多模态模型**，支持以下模态的输入与输出：

| 模态 | 支持情况 | 说明 |
|------|----------|------|
| **文本** | 完全支持 | 输入输出均完全支持 |
| **图像** | 支持输入 | 支持图像理解和分析 |
| **语音** | 支持输入/输出 | 语音识别（ASR）和语音合成（TTS） |
| **视频** | 部分支持 | 通过海螺AI 提供视频生成服务 |

多模态输入通过在请求中传入图像 URL 或 Base64 编码实现，模型能够理解图像内容并进行问答、描述、分析等任务。

## 3. 开放平台与应用程序

### 3.1 MiniMax API 平台

MiniMax 提供官方 API 服务，开发者可以通过 API 将 MiniMax 的模型能力集成到自己的应用中。

**API 平台地址**：https://api.minimax.chat

**主要特点**：

- RESTful API 设计，支持 HTTP/HTTPS 调用
- 支持 WebSocket 连接用于流式输出（SSE）
- 提供多种模型的统一接入
- 按 token 用量计费

### 3.2 MiniMax 对话应用

MiniMax 对话应用是 MiniMax 面向普通用户的直接访问入口，类似于 ChatGPT 的网页端和移动端产品。

**访问方式**：

- 网页版：https://chat.minimax.chat
- iOS App：App Store 搜索 "MiniMax" 或 "MiniMax Chat"
- Android App：通过应用商店或官网下载

**功能特点**：

- 支持多轮对话
- 支持图像上传和理解
- 支持语音输入和输出
- 提供多种对话模式（通用对话、创意写作、代码助手等）

### 3.3 海螺AI（Hailuo AI）

海螺AI 是 MiniMax 旗下的 AI 视频生成应用，主打高质量的文生视频和图生视频能力。

**访问方式**：

- 网页版：https://hailuoai.video
- iOS/Android App

**核心功能**：

- 文生视频（Text-to-Video）：根据文字描述生成视频
- 图生视频（Image-to-Video）：将静态图片转化为视频
- 视频风格转换
- 支持多种分辨率和时长选择

## 4. 如何使用

### 4.1 API 调用方式

MiniMax API 支持两种调用方式：同步调用和流式调用。

#### 4.1.1 同步调用示例

以下是基于 Python 的同步 API 调用示例：

```python
import requests
import json

# API 配置
API_KEY = "YOUR_API_KEY"  # 从 MiniMax 控制台获取
BASE_URL = "https://api.minimax.chat/v1"

def chat_completion(messages, model="abab6.5s"):
    """
    同步调用 MiniMax 对话接口
    """
    url = f"{BASE_URL}/text/chatcompletion_v2"

    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    }

    payload = {
        "model": model,
        "messages": messages,
        "temperature": 0.7,
        "max_tokens": 2048
    }

    response = requests.post(url, headers=headers, json=payload)
    return response.json()

# 使用示例
messages = [
    {"role": "system", "content": "你是一个专业的AI助手。"},
    {"role": "user", "content": "请介绍一下MiniMax MoE架构的优势。"}
]

result = chat_completion(messages)
print(result)
```

#### 4.1.2 流式调用示例

以下是基于 Python 的流式输出（SSE）调用示例：

```python
import requests
import json

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://api.minimax.chat/v1"

def chat_completion_stream(messages, model="abab6.5s"):
    """
    流式调用 MiniMax 对话接口（SSE）
    """
    url = f"{BASE_URL}/text/chatcompletion_v2"

    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    }

    payload = {
        "model": model,
        "messages": messages,
        "temperature": 0.7,
        "max_tokens": 2048,
        "stream": True  # 开启流式输出
    }

    response = requests.post(url, headers=headers, json=payload, stream=True)

    for line in response.iter_lines():
        if line:
            # 解析 SSE 格式数据
            data = line.decode('utf-8')
            if data.startswith('data: '):
                data = data[6:]  # 去掉 "data: " 前缀
                if data == '[DONE]':
                    break
                try:
                    chunk = json.loads(data)
                    if 'choices' in chunk and len(chunk['choices']) > 0:
                        delta = chunk['choices'][0].get('delta', {})
                        if 'content' in delta:
                            print(delta['content'], end='', flush=True)
                except json.JSONDecodeError:
                    continue

# 使用示例
messages = [
    {"role": "user", "content": "用一首诗描述春天的景色"}
]

chat_completion_stream(messages)
```

#### 4.1.3 支持函数调用（Function Calling）

```python
import requests

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://api.minimax.chat/v1"

def chat_with_functions():
    """
    使用 Function Calling 功能
    """
    url = f"{BASE_URL}/text/chatcompletion_v2"

    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    }

    # 定义可调用的函数
    tools = [
        {
            "type": "function",
            "function": {
                "name": "get_weather",
                "description": "获取指定城市的天气信息",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "city": {
                            "type": "string",
                            "description": "城市名称"
                        }
                    },
                    "required": ["city"]
                }
            }
        }
    ]

    messages = [
        {"role": "user", "content": "北京今天天气怎么样？"}
    ]

    payload = {
        "model": "abab6.5s",
        "messages": messages,
        "tools": tools,
        "temperature": 0.7
    }

    response = requests.post(url, headers=headers, json=payload)
    result = response.json()

    print("模型响应：")
    print(json.dumps(result, ensure_ascii=False, indent=2))

    return result

# 示例输出
# 模型会根据对话内容自动判断是否需要调用 get_weather 函数
result = chat_with_functions()
```

### 4.2 SDK 使用

MiniMax 提供官方 Python SDK，简化 API 调用流程。

#### 4.2.1 安装

```bash
pip install minimax-python
```

#### 4.2.2 Python SDK 使用示例

```python
from minimax import MiniMax

# 初始化客户端
client = MiniMax(api_key="YOUR_API_KEY")

# 对话补全（同步）
response = client.chat.completion(
    model="abab6.5s",
    messages=[
        {"role": "system", "content": "你是一个乐于助人的AI助手。"},
        {"role": "user", "content": "解释一下什么是MoE架构。"}
    ],
    temperature=0.7,
    max_tokens=1024
)

print(response.choices[0].message.content)

# 对话补全（流式）
for chunk in client.chat.completion_stream(
    model="abab6.5s",
    messages=[
        {"role": "user", "content": "写一段Python代码实现快速排序"}
    ],
    temperature=0.7
):
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
```

### 4.4 本地部署

## 6. 优缺点分析

### 6.1 主要优势

| 优势 | 说明 |
|------|------|
| **MoE 架构高效** | 采用混合专家架构，推理成本低，性价比高 |
| **长上下文支持** | 支持 256K tokens 的超长上下文，适合长文处理场景 |
| **多模态能力** | 支持文本、图像、语音等多种模态 |
| **中文能力强** | 在中文理解、生成、对话方面表现优异 |
| **Function Calling** | 支持函数调用，便于构建 AI 应用 |
| **流式输出** | 支持 SSE 流式输出，提升用户体验 |

### 6.2 主要劣势

| 劣势 | 说明 |
|------|------|
| **非开源** | 模型不开源，无法进行本地部署和自定义训练 |
| **海外可用性有限** | 主要面向中国市场，海外节点可能较少 |
| **生态相对年轻** | 相比 OpenAI、Google 等，开发者社区和工具链还在建设中 |
| **视频生成需另用产品** | 视频生成能力通过独立产品（海螺AI）提供，非 API 直接支持 |

### 6.3 与其他国产模型对比

| 维度 | MiniMax | DeepSeek | Qwen | Kimi |

| **架构** | MoE | MoE | Dense+MoE | MoE |

## 8. API 端点与参数详解

### 8.1 主要 API 端点

#### 8.1.1 文本对话

```
POST https://api.minimax.chat/v1/text/chatcompletion_v2
```

**请求头**：

| 参数 | 必填 | 说明 |
|------|------|------|
| Authorization | 是 | Bearer {API_KEY} |
| Content-Type | 是 | application/json |

**请求体参数**：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| model | string | 是 | - | 模型名称，如 "abab6.5s" |
| messages | array | 是 | - | 对话消息列表 |
| temperature | float | 否 | 0.7 | 采样温度，范围 0-2 |
| max_tokens | int | 否 | 2048 | 最大输出 tokens 数 |
| top_p | float | 否 | 0.95 | Top-p 采样参数 |
| stream | bool | 否 | false | 是否开启流式输出 |
| stop | string/array | 否 | - | 停止生成的标记 |
| tools | array | 否 | - | 函数调用定义 |
| tool_choice | string | 否 | - | 函数调用选择策略 |

**Messages 格式**：

```json
[
  {"role": "system", "content": "系统提示词"},
  {"role": "user", "content": "用户消息"},
  {"role": "assistant", "content": "助手回复"},
  {"role": "user", "content": "用户的下一条消息"}
]
```

**响应格式**：

```json
{
  "id": "chatcmpl-xxxxx",
  "model": "abab6.5s",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "模型生成的回复内容"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 100,
    "completion_tokens": 200,
    "total_tokens": 300
  },
  "created": 1712000000
}
```

#### 8.1.2 流式响应格式

SSE 格式，每条消息格式如下：

```
data: {"id":"chatcmpl-xxx","choices":[{"index":0,"delta":{"content":"部"}}],"usage":{"prompt_tokens":10,"completion_tokens":1,"total_tokens":11},"created":1712000000}

data: {"id":"chatcmpl-xxx","choices":[{"index":0,"delta":{"content":"分"}}],"usage":{"prompt_tokens":10,"completion_tokens":2,"total_tokens":12},"created":1712000000}

data: [DONE]
```

#### 8.1.3 图像理解

```
POST https://api.minimax.chat/v1/text/chatcompletion_v2
```

图像通过消息内容中传入图片 URL 或 Base64 编码：

```json
{
  "model": "abab6.5s",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "这张图片里有什么？"},
        {"type": "image_url", "image_url": {"url": "https://example.com/image.jpg"}}
      ]
    }
  ]
}
```

或使用 Base64：

```json
{
  "model": "abab6.5s",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "这张图片里有什么？"},
        {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,/9j/4AAQ..."}}
      ]
    }
  ]
}
```

#### 8.1.4 文本嵌入（Embedding）

```
POST https://api.minimax.chat/v1/text/embeddings
```


**错误响应格式**：

## 9. 最佳实践与注意事项

### 9.1 提示词工程建议

1. **明确任务目标**：在 system prompt 中清晰定义角色和能力边界
2. **结构化输出**：需要 JSON 输出时，使用 JSON Mode 或明确指定格式
3. **分步引导**：复杂任务拆分为多个步骤，避免一次性要求过多
4. **温度参数选择**：
   - 0.0-0.3：确定性任务（翻译、代码）
   - 0.5-0.7：通用对话
   - 0.8-1.0：创意任务

### 9.2 成本优化建议

1. **合理设置 max_tokens**：避免设置过大，预估合理范围
2. **对话历史压缩**：长对话可使用摘要或重新开启对话
3. **选择合适模型**：简单任务使用 abab6.5s，复杂任务使用 abab6.5
4. **批量处理**：多个独立任务合并一次 API 调用（如果适用）

### 9.3 安全性注意事项

1. **API Key 安全**：不要在前端代码中暴露 API Key，使用后端代理
2. **内容审核**：对用户输入和模型输出进行必要的内容审核
3. **速率限制**：遵守 API 调用频率限制，避免触发限流
