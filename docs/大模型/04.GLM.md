# 智谱AI GLM 大模型

## 1. 概述与历史

### 1.1 公司简介

智谱AI（Zhipu AI，全称：北京智谱华章科技有限公司）成立于2019年，是一家专注于大型语言模型研发的中国人工智能科技公司。公司由清华大学KEG实验室技术团队孵化成立，总部位于北京。智谱AI是国内最早从事大模型研究和商业化应用的企业之一。

智谱AI的核心团队在自然语言处理领域拥有深厚的技术积累，公司致力于打造安全、可靠、可解释的人工智能模型，其GLM（General Language Model）系列模型已成为国内最具影响力的大模型之一。

### 1.2 GLM系列发展历程

| 时间 | 版本 | 重要里程碑 |
|------|------|-----------|
| 2022年 | GLM-130B | 开源130B参数双语模型，刷新多项榜单 |
| 2023年 | GLM-4 | 整体性能逼近GPT-4，支持更长上下文 |
| 2024年初 | GLM-4V | 推出视觉多模态版本，支持图像理解 |
| 2024年中 | GLM-4-Plus | 增强版发布，性能全面提升 |
| 2024年 | GLM-4-AllTools | 支持工具调用和函数执行 |
| 2024年 | GLM-4-Long | 支持超长上下文（128K/1M tokens） |
| 2025年 | GLM-4.5 | 性能大幅提升，成本进一步降低 |
| 2025年 | GLM-4.5-Vision | 增强多模态能力 |
| 2026年 | GLM-4.7 | 最新版本，综合能力显著提升 |

### 3.2 SDK使用

#### 3.2.1 Python SDK

```bash
pip install zhipuai
```

```python
from zhipuai import ZhipuAI

client = ZhipuAI(api_key="your_api_key")

# 同步调用
response = client.chat.completions.create(
    model="glm-4",
    messages=[
        {"role": "user", "content": "请介绍一下北京的历史"}
    ],
    temperature=0.7,
    max_tokens=2000
)

print(response.choices[0].message.content)

# 异步调用
import asyncio

async def async_call():
    response = await client.chat.completions.acreate(
        model="glm-4",
        messages=[
            {"role": "user", "content": "解释量子计算的基本原理"}
        ]
    )
    return response

result = asyncio.run(async_call())
```

#### 3.2.2 Java SDK

```java
import cn.zhipuai.ZhipuAI;
import cn.zhipuai.model.*;

public class Main {
    public static void main(String[] args) {
        ZhipuAI client = new ZhipuAI("your_api_key");

        ChatCompletionRequest request = ChatCompletionRequest.builder()
            .model("glm-4")
            .messages(Messages.ofUser("解释机器学习和深度学习的区别"))
            .temperature(0.7f)
            .maxTokens(1000)
            .build();

        ChatCompletionResponse response = client.createChatCompletion(request);
        System.out.println(response.getChoices().get(0).getMessage().getContent());
    }
}
```

#### 3.3.3 使用Transformers加载

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

model_name = "THUDM/chatglm3-6b"

tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    trust_remote_code=True,
    device_map="auto"
)

input_text = "解释人工智能的发展历程"
inputs = tokenizer(input_text, return_tensors="pt").to(model.device)
outputs = model.generate(**inputs, max_new_tokens=500)
response = tokenizer.decode(outputs[0], skip_special_tokens=True)
print(response)
```

4. **批量处理**：合并多个请求降低开销

| 参数名 | 类型 | 说明 |
|--------|------|------|
| model | string | 模型名称，如 "glm-4"、"glm-4v-plus" |
| messages | array | 对话消息数组 |

**可选参数：**

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| temperature | float | 0.95 | 采样温度，0-2之间，越高越随机 |
| top_p | float | 0.7 | 核采样概率 |
| max_tokens | integer | 1024 | 最大生成token数 |
| stream | boolean | false | 是否流式返回 |
| stop | string/array | null | 停止生成的条件 |
| seed | integer | null | 随机种子，用于复现 |
| user | string | null | 用户标识 |

**messages格式：**

```json
{
  "messages": [
    {
      "role": "system",
      "content": "系统提示词"
    },
    {
      "role": "user",
      "content": "用户消息"
    },
    {
      "role": "assistant",
      "content": "助手回复"
    }
  ]
}
```

**多模态消息格式（GLM-4V系列）：**

```json

  "messages": [
```
