产品设计 Skills

产品经理日常工作所需的 Skills 工具集，帮助提升产品设计效率。

目录

分组Skill核心定位产出物
撰写PRD01.brainstorming发散思路、收敛方案、识别风险架构设计文档
撰写PRD02.write-a-prd深度访谈、结构化输出、逻辑验证PRD 文档
撰写PRD03.writing-plans细粒度拆解、验收标准、任务清单可执行实施计划
PRD自审07.prd-review-1.0.1系统化评审、质量检查、完整性校验评审纪要
增长与数据04.ab-test-setup统计严格的 A/B 实验方案设计实验设计文档
增长与数据05.analytics-tracking事件命名规范、追踪计划、隐私合规埋点方案
增长与数据06.onboarding-cro用户激活优化、Aha 时刻设计激活流程方案
实战经验08.实战记录实际项目中的使用经验与心得复盘文档

分组说明

撰写PRD

从想法到可执行计划的三阶段工作流：

想法 → brainstorming → write-a-prd → writing-plans → 开发执行
Skill工作方式适用场景
brainstorming一次只问一个问题，逐步厘清需求需求模糊、方向不确定
write-a-prd深度访谈式，追问每个设计分支方案确认后、输出结构化 PRD
writing-plans拆解为 2-5 分钟的微任务，含验收标准PRD 确认后、与研发对齐

PRD自审

在进入开发阶段前，对 PRD 进行系统化质量检查，确保需求达到"可商业化、可技术落地、可验收、可衡量"标准。

增长与数据

Skill核心价值典型场景
ab-test-setup设计统计严格的实验页面转化率优化、功能灰度
analytics-tracking建立决策导向的数据体系埋点方案、增长实验追踪
onboarding-cro找到用户 Aha 时刻激活率提升、邮件触达设计

实战经验

记录在实际项目中使用 Skills 的经验与心得，包括踩坑记录、最佳实践等。

推荐使用流程

标准流程：01 → 02 → 03 → 07

  1. brainstorming：先用头脑风暴厘清方向和方案
  2. write-a-prd：确认方案后撰写完整 PRD
  3. writing-plans：最后拆解为可执行任务计划
  4. prd-review：评审 PRD，确保质量达标

安装

所有 Skills 均通过以下命令安装：

npx skills add <author>/<repo> --skill <skill-name>
